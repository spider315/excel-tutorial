# 專案排程甘特圖 — AI 指令教學

> **難度**：進階
> **適用對象**：專案經理、主管、團隊領導
> **學習目標**：用 AI 自動產生甘特圖，追蹤專案進度與里程碑

---

## 📌 課程情境

**你是一位專案經理**，負責管理一個跨部門的 ERP 系統升級專案：

1. 專案有 20+ 個任務，分成 5 個階段
2. 每週要向老闆報告專案進度
3. 需要一張清楚的甘特圖，標示已完成/進行中/延遲
4. 還要追蹤里程碑是否如期達成

用 Excel 手動畫甘特圖太花時間，現在用 AI 幾分鐘就能產生專業的甘特圖報表。

## 🎯 Prompt 設計原則

### 原則 1：甘特圖結構要明確

```
❌ 不好的寫法：
幫我畫甘特圖

✅ 好的寫法：
產生甘特圖 Excel：
- 前 7 欄：任務編號、任務名稱、負責人、開始日期、結束日期、進度、狀態
- 第 8 欄開始：日期欄位（以週為單位）
- 每一列代表一個任務
- 根據開始/結束日期，在對應的日期欄填入顏色（條狀圖）
```

### 原則 2：顏色編碼要定義清楚

```
❌ 不好的寫法：
不同狀態用不同顏色

✅ 好的寫法：
甘特條顏色定義：
- 已完成：深綠色 #548235
- 進行中（已完成部分）：淺綠色 #70AD47
- 進行中（未完成部分）：最淺綠色 #A9D08E
- 延遲：紅色 #C00000

狀態欄顏色定義：
- 已完成：淺綠底 #C6EFCE
- 進行中：淺黃底 #FFEB9C
- 未開始：灰底 #DDDDDD
- 延遲：淺紅底 #FFC7CE
```

### 原則 3：日期處理要精確

```
❌ 不好的寫法：
用日期畫圖

✅ 好的寫法：
日期欄位處理：
1. 取得所有任務的最早開始日期與最晚結束日期
2. 以週為單位建立欄位（每欄代表一週）
3. 欄位標題格式：MM/DD（如 03/01）
4. 任務的甘特條：從開始日期對應的週 到 結束日期對應的週
```

### 原則 4：進度計算要視覺化

```
❌ 不好的寫法：
顯示進度

✅ 好的寫法：
進度視覺化：
- 假設任務 T003 工期 3 週，進度 66%
- 第 1-2 週的儲存格填入「已完成」顏色
- 第 3 週的儲存格填入「計畫中」顏色
- 計算公式：完成週數 = 總週數 × (進度% / 100)
```

## 📝 完整 Prompt 範本

### Prompt 1：任務資料清理

```
【角色】你是專案管理專家，熟悉甘特圖製作與 Python 程式設計

【情境】
- 輸入檔案：raw/project_tasks.xlsx（專案任務清單）
- 輸出目錄：output/

【任務】資料清理

1. 日期格式統一為 YYYY/MM/DD
   - 2025-03-14 → 2025/03/14
   - 25/04/20 → 2025/04/20

2. 負責人名字去除前後空白

3. 進度與狀態一致性檢查：
   - 進度 = 100% 但狀態 ≠ 已完成 → 修正狀態為「已完成」
   - 進度 > 0% 且 < 100% 但狀態 = 未開始 → 修正狀態為「進行中」

4. 日期邏輯檢查：
   - 若開始日期 > 結束日期 → 交換兩個日期

5. 重新計算工期：
   - 工期(天) = 結束日期 - 開始日期

【輸出】
- cleaned_tasks.xlsx：清理後的任務清單
- cleaning_log.xlsx：清理日誌

【驗證】顯示清理修正的總筆數
```

### Prompt 2：甘特圖產生

```
【角色】你是專案管理專家

【情境】
- 輸入：output/cleaned_tasks.xlsx
- 輸出：output/gantt_chart.xlsx

【任務】產生甘特圖樣式的 Excel

工作表結構：
- 欄 A-G：任務編號、任務名稱、負責人、開始日期、結束日期、進度、狀態
- 欄 H 起：日期欄位（以週為單位，從最早開始日期到最晚結束日期）
- 每一列對應一個任務

甘特條繪製邏輯：
1. 根據任務的開始日期和結束日期，決定填色範圍
2. 若該週在任務範圍內，則填入顏色
3. 顏色依狀態決定：
   - 已完成 → 深綠 #548235
   - 延遲 → 紅色 #C00000
   - 進行中：
     - 已完成週數 → 淺綠 #70AD47
     - 未完成週數 → 最淺綠 #A9D08E
   - 未開始 → 最淺綠 #A9D08E

狀態欄條件格式：
- 已完成 → #C6EFCE
- 進行中 → #FFEB9C
- 未開始 → #DDDDDD
- 延遲 → #FFC7CE

【格式】
- 標題列：深藍底 #2F5496、白字
- 日期欄寬度：6
- 任務欄寬度：20
```

### Prompt 3：進度追蹤報表

```
【角色】你是專案管理專家

【情境】
- 輸入：output/cleaned_tasks.xlsx
- 輸出：output/progress_report.xlsx

【任務】產生進度追蹤報表

Sheet 1「階段進度」：
- 列：每個階段（Phase 1-5）
- 欄：階段代碼、階段名稱、任務數、平均進度(%)、總工期(天)
- 平均進度 = 該階段所有任務進度的 AVG

Sheet 2「人員工作量」：
- 列：每位負責人
- 欄：負責人、任務數、總工期(天)、平均進度(%)
- 依總工期降冪排序

Sheet 3「狀態統計」：
- 列：已完成、進行中、未開始、延遲
- 欄：狀態、數量

【格式】
- 標題列：深藍底、白字
- 數字欄：千分位或一位小數
```

### Prompt 4：里程碑追蹤

```
【角色】你是專案管理專家

【情境】
- 輸入：raw/milestones.xlsx
- 輸出：output/milestone_tracking.xlsx

【任務】里程碑風險追蹤

1. 讀取里程碑資料（里程碑編號、里程碑名稱、目標日期、達成狀態）

2. 計算剩餘天數：
   - 剩餘天數 = 目標日期 - 今天

3. 評估風險等級：
   - 已達成 → 「無」
   - 剩餘天數 < 0 → 「高（已逾期）」
   - 剩餘天數 < 7 → 「中（7天內）」
   - 其他 → 「低」

4. 條件格式：
   - 風險「高」→ 淺紅底 #FFC7CE
   - 風險「中」→ 淺黃底 #FFEB9C

【輸出】milestone_tracking.xlsx
```

## 💡 常見錯誤與修正

### 錯誤 1：甘特條沒有出現

**問題**：日期欄全部空白

**原因**：日期格式不一致，導致比對失敗

**修正**：先統一日期格式為 `YYYY/MM/DD`，再進行甘特條繪製

### 錯誤 2：進度百分比顯示錯誤

**問題**：進度欄顯示 0.5 而非 50%

**原因**：讀取時被轉為小數

**修正**：在 Prompt 中指定「進度欄顯示為百分比格式（如 50%）」

### 錯誤 3：週次計算錯誤

**問題**：甘特條位置偏移

**原因**：週次計算沒有對齊週一

**修正**：使用 `timedelta(days=7)` 固定每週間隔，不用 `isocalendar()`

## 🚀 進階練習

1. **關鍵路徑分析**：找出影響專案完成時間的關鍵任務路徑

2. **資源衝突檢測**：檢查同一人是否被分配到同時間的多個任務

3. **自動延遲標記**：若今天已超過結束日期但進度 < 100%，自動標記為「延遲」

4. **Baseline 比較**：比較原始計畫與目前進度的差異
