# AI Prompt 教學講義：用 AI 完成 HR 資料處理任務

> 適用對象：人資行政人員（不需要程式背景）
> 教學目標：學會撰寫有效的 AI Prompt，讓 AI 幫你自動完成 Excel 資料清理、比對與報表產出

---

## 一、前言：為什麼人資行政可以用 AI 自動化？

### 日常 HR 作業的痛點

身為人資夥伴，你一定對這些場景不陌生：

- **多檔案來回比對**：退休名冊、健保轉出清單、給付通知分別由不同單位提供，格式不統一，每次都要人工逐筆核對
- **格式不一致**：有人填 `2025/06/30`，有人填 `2025-06-30`；有人身分證字號打小寫，有人前後多了空白
- **重複與矛盾難發現**：同一位員工在兩份檔案中姓名不同、同一員工編號出現兩次，靠肉眼很難全部抓到
- **人工出錯風險高**：手動複製貼上產生通知信，一不小心就貼錯金額或日期

這些工作不需要創意或判斷力，但需要極高的細心度——這正是 AI 最擅長的事。

### AI 程式碼工具可以怎麼幫忙？

現在有許多 AI 工具可以幫你「用講的就寫出程式」：

| 工具名稱 | 特色 |
|----------|------|
| **Claude Code** | Anthropic 推出的 AI 程式碼助手，可在網頁版或終端機使用 |
| **Codex CLI** | OpenAI 推出的命令列工具，適合自動化腳本 |
| **Gemini CLI** | Google 推出的命令列工具 |
| **Cursor / Windsurf** | 內建 AI 的程式碼編輯器，有圖形介面 |

這些工具的共同點是：**你只需要用自然語言描述你要什麼，AI 就會幫你寫出程式並執行。**

### 核心觀念

> **你不需要會寫程式，只需要會描述你要什麼。**

這份教學要教你的，就是「如何把你腦中的需求，轉換成 AI 能精準執行的指令」——這個指令就叫做 **Prompt**。

## 二、完整案例說明

### 情境描述

本案例模擬一個真實的 HR 退休作業場景。我們有三份 Excel 檔案，分別來自不同業務單位：

| 檔案 | 說明 | 欄位 |
|------|------|------|
| `retirement_roster.xlsx`<br>退休名冊 | 30 筆，記錄即將退休的員工資料 | 員工編號、姓名、身分證字號、出生日期、到職日、預計退休日、部門、職等 |
| `nhi_transfer_list.xlsx`<br>健保轉出清單 | 25 筆，記錄健保轉出資訊 | 員工編號、姓名、身分證字號、轉出日期、轉出原因、投保金額、眷屬人數 |
| `payment_notification.xlsx`<br>給付通知資料 | 20 筆，記錄退休給付資訊 | 員工編號、姓名、給付類型、應付金額、銀行代碼、帳號、發放日期、備註 |

三份檔案透過「**員工編號**」互相關聯，部分員工會同時出現在多份檔案中。

### 原始資料中刻意埋入的問題

為了教學示範，這些資料中刻意混入了以下類型的問題：

**格式錯誤：**
- 身分證字號少一碼（只有 9 碼或 8 碼）
- 身分證字號英文字母用了小寫
- 日期格式不統一（有的用 `/` 分隔，有的用 `-`）
- 姓名前後有多餘空白
- 備註欄位含有零寬字元、BOM 等亂碼

**邏輯矛盾：**
- 預計退休日早於到職日（不合理）
- 轉出日期為空值（應該要有）

**跨檔不一致：**
- 同一員工編號在退休名冊和健保清單中姓名不同
- 同一份檔案中出現重複的員工編號

### 目標產出

我們要用 AI 完成以下工作流程：

```
原始髒資料（3 份 Excel）
    ↓ 任務一：自動清理
清理後資料（3 份 Excel）
    ↓ 任務二：異常偵測
異常報告（1 份 Excel，4 個工作表）
    ↓ 任務三：摘要報表 + 通知信
摘要報表（1 份 Excel，3 個工作表）+ 通知信（1 份文字檔）
```

## 三、Prompt 撰寫教學（重點中的重點）

這是本教學最核心的部分。我們會用「拆解式教學法」，帶你一步步看懂 Prompt 是怎麼寫出來的。

### 3.1 Prompt 設計的五大原則

#### 原則一：明確指定輸入與輸出

告訴 AI 要讀什麼檔案、結果存到哪裡。

```
❌ 不好的寫法：
幫我處理 HR 資料

✅ 好的寫法：
請讀取 hr_demo/raw/ 目錄下的三份 Excel 檔案：
- retirement_roster.xlsx（退休名冊）
- nhi_transfer_list.xlsx（健保轉出清單）
- payment_notification.xlsx（給付通知資料）

處理結果請輸出到 hr_demo/output/ 目錄。
```

**為什麼？** AI 不知道你的檔案叫什麼名字、放在哪裡。你給的路徑越明確，AI 出錯的機會越小。

#### 原則二：分任務描述

把大任務拆成小任務，每個任務獨立描述。

```
❌ 不好的寫法：
把資料整理好然後產出報表

✅ 好的寫法：
任務一：自動清理資料格式（姓名去空白、日期統一、身分證轉大寫、備註去亂碼）
任務二：比對三份檔案，找出格式異常、邏輯矛盾、跨檔不一致，產出異常報告
任務三：合併三份檔案產出摘要報表，並為每位退休員工產生通知信
```

**為什麼？** 一次丟太多要求，AI 容易遺漏。拆成獨立任務，每個任務的範圍清楚，AI 才能逐一完成。

#### 原則三：具體列出檢查規則

不要用「檢查異常」這種模糊說法，要明確寫出每一條規則。

```
❌ 不好的寫法：
檢查有沒有錯誤

✅ 好的寫法：
格式異常檢查規則：
- 身分證字號必須是 1 碼大寫英文 + 9 碼數字
- 日期必須符合 YYYY-MM-DD 格式
- 投保金額不得為負數
- 應付金額不得為 0
- 銀行代碼必須是 3 碼數字
- 帳號長度必須在 10～14 碼之間
```

**為什麼？** 你心中的「錯誤」和 AI 理解的「錯誤」可能完全不同。把規則寫死，AI 才能跟你想的一樣。

#### 原則四：提供輸出格式範本

想要什麼格式的輸出，直接給模板。

```
❌ 不好的寫法：
產一封通知信

✅ 好的寫法：
通知信模板：

【退休離職通知】
{姓名} 先生/女士 您好：
感謝您在本公司 {部門} 服務多年。您的退休生效日為 {預計退休日}。
相關退休給付金額為新台幣 {應付金額} 元整，將於 {發放日期} 撥入您指定帳戶。
健保將於 {轉出日期} 辦理轉出。
如有任何疑問，請洽人力資源部。
```

**為什麼？** AI 很擅長「照著模板填空」，但不擅長「猜你想要什麼格式」。給了模板，產出就會完全符合你的期待。

#### 原則五：說明資料關聯

告訴 AI 三份檔案之間的關係。

```
❌ 不好的寫法：
把三份資料合在一起

✅ 好的寫法：
三份檔案透過「員工編號」欄位關聯。
退休人員總表只包含「三份檔案都有出現」的員工（inner join）。
以退休名冊為主要資料來源，補上健保清單的轉出日期、投保金額，以及給付通知的給付類型、應付金額。
```

**為什麼？** 「合在一起」可以是 inner join、left join、full outer join……不同的合併方式結果完全不同。你要明確告訴 AI 用哪種。

### 3.2 逐段拆解 Prompt 2（資料處理）

以下是我們實際使用的 Prompt，逐段拆解並說明「為什麼要這樣寫」。

---

**【第一段：指定腳本與路徑】**

```
請在 hr_demo/ 目錄下建立 process_data.py，
讀取 hr_demo/raw/ 的三份原始 Excel，
處理結果輸出到 hr_demo/output/ 目錄。
```

> **為什麼這樣寫？**
> 這段話做了三件事：（1）指定腳本檔案名稱與位置、（2）指定輸入資料的目錄、（3）指定輸出目錄。AI 有了這些資訊，就知道要讀什麼、寫到哪裡，不會把檔案亂存。
>
> **教學重點：** 永遠在 Prompt 的開頭就把「讀哪裡、寫哪裡」講清楚。

---

**【第二段：任務一 — 清理規則】**

```
任務一：自動清理資料格式

清理規則：
1. 姓名：去除前後多餘空白（例如 " 王淑芬" → "王淑芬"）
2. 日期：統一格式為 YYYY-MM-DD（原始資料可能使用 / 分隔，如 2025/06/30 → 2025-06-30）
3. 身分證字號：英文字母統一轉大寫（如 a123456789 → A123456789）
4. 備註：移除零寬字元（\u200b）、BOM（\ufeff）等特殊控制字元，僅保留正常中英文與標點

輸出三份清理後的檔案：
- cleaned_retirement_roster.xlsx
- cleaned_nhi_transfer_list.xlsx
- cleaned_payment_notification.xlsx
```

> **為什麼這樣寫？**
> 每條清理規則都搭配了**具體的範例**（如 `" 王淑芬" → "王淑芬"`），讓 AI 不用猜測「去除空白」到底是什麼意思。備註的清理甚至列出了具體的 Unicode 字元代碼，確保 AI 知道要移除哪些字元。
>
> **教學重點：** 清理規則不能模糊。每條規則至少要有一個「Before → After」的範例，AI 才能準確理解你要的轉換方式。

---

**【第三段：任務二 — 異常比對】**

```
任務二：產出異常報告 anomaly_report.xlsx

Sheet 1「格式異常」
欄位：來源檔案, 列號, 欄位名稱, 原始值, 問題描述
檢查規則：
- 身分證字號不符合「1碼大寫英文+9碼數字」格式
- 日期不符合 YYYY-MM-DD 格式
- 投保金額為負數
- 應付金額為 0
- 銀行代碼非 3 碼數字
- 帳號長度不在 10-14 碼之間

Sheet 2「邏輯異常」
欄位：來源檔案, 列號, 問題描述, 相關欄位值
檢查規則：
- 預計退休日早於到職日
- 轉出日期為空值

Sheet 3「跨檔比對異常」
欄位：員工編號, 問題描述, 檔案一值, 檔案二值
檢查規則：
- 同一員工編號在不同檔案中姓名不一致
- 同一員工編號重複出現

Sheet 4「重複資料」
欄位：來源檔案, 員工編號, 出現次數, 相關列號
```

> **為什麼這樣寫？**
> 這段做了兩件關鍵的事：
> 1. **指定每個工作表的名稱和欄位**：AI 不用自己決定報告格式，直接照著你的結構輸出
> 2. **逐條列出檢查規則**：每條規則都是具體的、可判斷的條件（不是「找出異常」這種模糊說法）
>
> **教學重點：** 當你需要多工作表的 Excel 時，一定要逐一寫出每個 Sheet 的名稱與欄位。這是最容易被遺漏、也最容易出錯的地方。

---

**【第四段：任務三 — 摘要報表】**

```
任務三：產出摘要報表 summary_report.xlsx

Sheet 1「總覽儀表板」：
- 各檔案資料筆數統計
- 各類型異常數量統計（格式異常、邏輯異常、跨檔比對異常、重複資料）
- 清理前後的資料品質對比（清理前異常率 vs 清理後異常率）

Sheet 2「退休人員總表」：
- 三份檔案以員工編號做 inner join（只保留三份都有的員工）
- 欄位：員工編號、姓名、身分證字號、部門、職等、預計退休日、轉出日期、投保金額、給付類型、應付金額

Sheet 3「給付明細」：
- 按部門與給付類型彙總總金額
- 列出全體平均給付金額
```

> **為什麼這樣寫？**
> 這段直接描述了「你想看到的報表長什麼樣」。特別注意「inner join」這個關鍵詞——它明確告訴 AI 只要保留三份檔案都有出現的員工，不要保留只出現在一兩份檔案的人。
>
> **教學重點：** 寫報表需求時，把自己想像成在跟一個新進同事說明「這份報表要呈現什麼」，你會怎麼說？那就是你的 Prompt。

---

**【第五段：通知信 — 給模板】**

```
同時產出 notification_letters.txt：
對退休人員總表中的每位員工產生通知信，模板如下：

【退休離職通知】
{姓名} 先生/女士 您好：
感謝您在本公司 {部門} 服務多年。您的退休生效日為 {預計退休日}。
相關退休給付金額為新台幣 {應付金額} 元整，將於 {發放日期} 撥入您指定帳戶。
健保將於 {轉出日期} 辦理轉出。
如有任何疑問，請洽人力資源部。

格式要求：
- 日期使用「YYYY年MM月DD日」格式
- 金額加入千分位逗號
- 每封通知之間用 --- 分隔
```

> **為什麼這樣寫？**
> 提供了「填空式模板」，AI 只需要把 `{姓名}`、`{部門}` 等佔位符替換成實際資料。格式要求（日期用中文格式、金額加千分位）也一併寫清楚，避免 AI 用預設格式輸出。
>
> **教學重點：** 想要什麼格式就給什麼模板。用 `{欄位名稱}` 這種佔位符，AI 一看就知道要填什麼。

---

**【第六段：結尾 — 要求顯示結果】**

```
完成後列出 output 目錄的所有檔案，並顯示 summary_report.xlsx 中「總覽儀表板」的完整內容。
```

> **為什麼這樣寫？**
> 這句話讓你可以**立即驗證**產出是否正確。如果不加這句，AI 可能只會說「完成了」，你還要自己去開檔案確認。要求它直接顯示結果，你馬上就能看出有沒有問題。
>
> **教學重點：** 每次 Prompt 的結尾，都加上「請顯示/列出結果」，這是最省時的驗證方式。

### 3.4 常見錯誤與改進範例

#### 錯誤一：需求太籠統

```
❌ 請幫我整理 Excel 資料

✅ 請讀取 hr_demo/raw/retirement_roster.xlsx，
   將姓名欄位去除前後空白，
   將日期欄位統一為 YYYY-MM-DD 格式，
   輸出為 hr_demo/output/cleaned_retirement_roster.xlsx
```

**問題：**「整理」可以是排序、篩選、去重複、改格式……AI 不知道你要哪一種。

---

#### 錯誤二：檢查標準不明

```
❌ 檢查有沒有錯誤

✅ 檢查以下格式異常：
   - 身分證字號必須符合「1碼大寫英文+9碼數字」
   - 投保金額不得為負數
   - 銀行代碼必須為 3 碼數字
   - 帳號長度必須在 10-14 碼之間
```

**問題：**「錯誤」的定義因人而異。你覺得是錯的，AI 可能覺得正常。把判斷標準寫成明確的規則。

---

#### 錯誤三：報表結構不清

```
❌ 產出報表

✅ 產出 anomaly_report.xlsx，包含四個工作表：
   Sheet 1「格式異常」欄位：來源檔案, 列號, 欄位名稱, 原始值, 問題描述
   Sheet 2「邏輯異常」欄位：來源檔案, 列號, 問題描述, 相關欄位值
   Sheet 3「跨檔比對異常」欄位：員工編號, 問題描述, 檔案一值, 檔案二值
   Sheet 4「重複資料」欄位：來源檔案, 員工編號, 出現次數, 相關列號
```

**問題：** AI 不知道你要幾個工作表、每個工作表叫什麼名字、要有哪些欄位。結果可能跟你想的完全不一樣。

---

#### 錯誤四：合併方式不明

```
❌ 把三份資料合在一起

✅ 三份檔案以「員工編號」欄位做 inner join，
   只保留三份檔案都有出現的員工。
   以退休名冊為主要資料來源。
```

**問題：**「合在一起」是要聯集（全部都要）還是交集（只要重疊的）？不說清楚，AI 只能猜。

---

#### 錯誤五：沒有驗證步驟

```
❌ （Prompt 結尾沒有任何驗證要求）

✅ 完成後列出 output 目錄的所有檔案，
   並顯示 summary_report.xlsx 中「總覽儀表板」的完整內容。
```

**問題：** 不要求顯示結果，你就無法立即確認產出是否正確，萬一有問題還要重新來過。

## 四、不同 AI 工具的使用方式

上面的 Prompt 可以在多種 AI 工具中使用，以下簡要說明各工具的操作方式：

### Claude Code（Web 版）

直接把 Prompt 貼入對話框即可。Claude Code 會自動建立檔案、執行 Python 腳本、顯示結果。

### Codex CLI

在終端機中執行：

```bash
codex "你的 Prompt 內容"
```

### Gemini CLI

在終端機中執行：

```bash
gemini "你的 Prompt 內容"
```

### Cursor / Windsurf

在編輯器中開啟 Composer（通常是 `Ctrl+I` 或 `Cmd+I`），將 Prompt 貼入對話框。

### 環境需求提醒

無論使用哪個工具，你的電腦都需要具備以下環境：

- **Python**（建議 3.8 以上版本）
- **pandas** 套件（處理表格資料）
- **openpyxl** 套件（讀寫 Excel 檔案）

如果不確定有沒有裝，可以請 AI 幫你檢查：

```
請檢查目前環境是否已安裝 Python、pandas 和 openpyxl，如果沒有請幫我安裝。
```

## 五、延伸應用

### 其他 HR 場景

本教學的 Prompt 撰寫技巧不只適用於退休作業，你可以套用到許多日常業務：

- **每月薪資核對**：比對薪資系統匯出檔與銀行轉帳明細，找出金額不一致的紀錄
- **加班時數統計**：讀取打卡紀錄，按部門、月份彙總加班時數，標記超時員工
- **年度考績彙整**：合併各部門主管的考績評分表，產出全公司排名與分布圖
- **勞健保級距調整**：比對員工薪資異動紀錄與目前投保級距，找出需要調整的人

### 如何修改 Prompt？

你只需要替換以下內容，就能把同一套 Prompt 結構用在不同的任務上：

1. **輸入檔案**：改成你實際的檔案名稱和欄位
2. **清理規則**：改成你需要的格式轉換
3. **檢查規則**：改成你業務上的驗證條件
4. **報表結構**：改成你想看到的統計方式
5. **通知模板**：改成你公司的公文格式

**鼓勵你動手試試！** 從一個小任務開始（例如：「幫我把這份 Excel 的日期欄位統一格式」），練習寫 Prompt，慢慢加入更多要求。每次多加一個任務，你就會越來越熟練。
