# 01 — 圖表類型選擇指南：用 AI 指令搭配正確的圖表

> 選對圖表類型，比花時間美化更重要。本篇教你如何根據資料特性，在 Prompt 中指定最合適的圖表。

---

## 一、圖表選型決策樹

在撰寫 Prompt 之前，先問自己三個問題：

1. **我想表達什麼？** — 趨勢、比較、佔比、分佈、關聯
2. **有幾組資料？** — 單系列 vs 多系列
3. **資料點有多少？** — 決定圖表的密度和可讀性

```
                    ┌─ 趨勢 → 折線圖 / 面積圖
                    │
                    ├─ 比較 ─┬─ 少量分類 → 柱狀圖
                    │        └─ 排名呈現 → 水平長條圖
你想表達什麼？ ─────┤
                    ├─ 佔比 ─┬─ ≤ 7 項 → 圓餅圖
                    │        └─ > 7 項 → 樹狀圖 / 堆疊柱狀圖
                    │
                    ├─ 關聯 → 散佈圖
                    │
                    └─ 多維度 → 雷達圖
```

---

## 二、各圖表類型詳解

### 2.1 折線圖（Line Chart）

**適用場景**：
- 時間序列資料（月度、季度、年度）
- 連續性變化趨勢
- 多組資料的趨勢比較

**Prompt 範本**：
```
用折線圖顯示 2025 年 1-12 月的營收趨勢。
- X 軸 = 月份（2025-01 到 2025-12）
- Y 軸 = 營收（千分位格式）
- 北區、中區、南區、東區各一條線
- 折線寬度 2.5pt，帶圓形資料標記
- 配色：北區=深藍、中區=暗紅、南區=深綠、東區=深金
```

**注意事項**：
- 資料點超過 20 個時，不要顯示資料標籤（會擠在一起）
- 多系列建議不超過 5 條線（否則難以區分）
- 若需要強調某條線，可以加粗或用虛線區分

---

### 2.2 柱狀圖（Column Chart）

**適用場景**：
- 不同類別之間的數值比較
- 分組比較（群組柱狀圖）
- 組成結構（堆疊柱狀圖）

**三種變體**：

| 變體 | grouping 值 | 適用場景 |
|------|------------|---------|
| 群組柱狀圖 | `"clustered"` | 並排比較各類別 |
| 堆疊柱狀圖 | `"stacked"` | 呈現總量中的組成 |
| 百分比堆疊 | `"percentStacked"` | 比較各類別的佔比結構 |

**Prompt 範本（群組）**：
```
用群組柱狀圖比較各區域的產品銷售營收。
- X 軸 = 區域（北區、中區、南區、東區）
- Y 軸 = 營收（元）
- 每個區域有 5 根柱子（5 種產品）
- 每種產品用不同顏色
```

**Prompt 範本（堆疊）**：
```
用堆疊柱狀圖顯示各季度的區域營收組成。
- X 軸 = 季度（Q1、Q2、Q3、Q4）
- Y 軸 = 營收（元）
- 每個柱子由 4 個區域的營收堆疊組成
- 從下到上：北區(深藍)、中區(暗紅)、南區(深綠)、東區(深金)
```

---

### 2.3 水平長條圖（Bar Chart）

**適用場景**：
- 排名呈現（業績排名、分數排名）
- 類別名稱較長時（橫向比直向更易讀）

**Prompt 範本**：
```
用水平長條圖顯示 15 位業務員的年度業績排名。
- 依實際業績升序排列（最高的在最上面）
- Y 軸 = 業務員姓名
- X 軸 = 實際業績（千分位格式）
- 所有長條用深藍色 #2F5496
```

**技巧**：升序排列讓最高值在最上方，符合閱讀習慣。

---

### 2.4 圓餅圖（Pie Chart）

**適用場景**：
- 顯示各部分佔整體的比例
- 項目數量 ≤ 7（超過就太擠了）

**Prompt 範本**：
```
用圓餅圖顯示市場份額分佈。
- 6 個品牌的市占率
- 顯示類別名稱 + 百分比（不顯示數值）
- 自訂各扇區顏色：
  自有品牌=#2F5496、品牌A=#C00000、品牌B=#548235、
  品牌C=#BF8F00、品牌D=#7030A0、其他=#A5A5A5
- 最大扇區稍微拉出（explode）
```

**注意事項**：
- ❌ 不要用圓餅圖比較兩組不同時期的資料 → 用柱狀圖
- ❌ 不要用圓餅圖顯示超過 7 個類別 → 合併小項為「其他」
- ❌ 不要用 3D 圓餅圖 → 扭曲比例，誤導讀者

---

### 2.5 環圈圖（Doughnut Chart）

**適用場景**：
- 與圓餅圖類似，但中央可放摘要數字
- 可呈現多圈（多層級）資料

**Prompt 範本**：
```
用環圈圖顯示市場份額，中央留白可供日後加入文字方塊。
- 顯示類別名稱 + 百分比
- 內圈大小 50%（holeSize = 50）
```

---

### 2.6 散佈圖（Scatter Chart）

**適用場景**：
- 分析兩個變數之間的關聯性
- 找出異常值（離群點）

**Prompt 範本**：
```
用散佈圖分析「總營收」與「毛利率」的關係。
- X 軸 = 總營收（千分位格式）
- Y 軸 = 毛利率（百分比格式）
- 每個產品一個點，顯示產品名稱標籤
- 資料點不連線（scatter only）
- 點的大小 = 8pt
```

---

### 2.7 雷達圖（Radar Chart）

**適用場景**：
- 多維度評估比較（3-8 個維度最佳）
- 能力雷達圖、滿意度分析

**Prompt 範本**：
```
用雷達圖比較 4 個區域在 5 個客戶滿意度維度的表現。
- 5 個軸：產品品質、售後服務、價格合理性、交貨速度、技術支援
- 4 條線：北區、中區、南區、東區
- 帶圓形資料標記 (type="marker")
- Y 軸範圍 0-5（分數制）
```

**注意事項**：
- 維度超過 8 個時，雷達圖會變得難以閱讀
- 各維度的量級應該一致（例如都是 1-5 分）

---

### 2.8 組合圖（Combo Chart）

**適用場景**：
- 兩個量級差異大的指標需要同時呈現
- 例如：金額（萬元級）+ 百分比

**Prompt 範本**：
```
用組合圖顯示每月預算與實際的比較：
- 主軸（柱狀圖）：預算金額（淺藍）、實際金額（深藍）
- 副軸（折線圖）：差異率（%），暗紅色折線
- 兩個 Y 軸：左邊=金額（千分位），右邊=百分比
- 折線的副軸 axId = 200
```

**關鍵提示**：在 Prompt 中務必提到「副軸」或「第二 Y 軸」，否則 AI 可能把所有資料放在同一軸上。

---

## 三、不適合用圖表的情況

不是所有資料都需要圖表。以下情況直接用表格更好：

| 情況 | 原因 |
|------|------|
| 只有 2-3 個數字 | 直接看數字比看圖更快 |
| 需要精確到小數的比較 | 圖表無法呈現精確數值差異 |
| 資料結構非常複雜 | 先用表格整理，再選圖表 |
| 資料量太少（< 3 個點） | 圖表沒有意義 |

---

## 四、圖表配色最佳實踐

### 色彩原則

1. **高對比度**：相鄰的顏色要能明顯區分
2. **色盲友好**：避免僅用紅/綠區分（加上圖案或標籤）
3. **一致性**：同一份報告中，同一個類別始終用同一種顏色
4. **不超過 8 色**：超過就很難區分

### 推薦配色方案

**商務藍色系**（適合正式報告）：
```
#2F5496, #5B9BD5, #8DB4E2, #B4C6E7
```

**多色系**（適合多類別比較）：
```
#2F5496, #C00000, #548235, #BF8F00, #7030A0, #00B0F0
```

**紅綠燈系**（適合達標/未達標）：
```
達標: #548235 (綠)
警示: #BF8F00 (黃)
未達: #C00000 (紅)
```
