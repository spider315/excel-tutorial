## 📌 課程情境

**你是一家電商公司的倉管主管**，每週要處理：

## 🎯 Prompt 設計原則

### 原則 1：庫存水位判斷要明確

```
❌ 不好的寫法：
標記庫存不足的商品

✅ 好的寫法：
庫存水位判斷規則：
- 可用庫存 = 總庫存 - 已預留數量
- 缺貨：可用庫存 ≤ 0
- 低於安全庫存：可用庫存 < 安全庫存量
- 庫存偏低：可用庫存 < 安全庫存量 × 1.5
- 正常：其他情況
```

### 原則 2：補貨計算公式要精確

```
❌ 不好的寫法：
計算需要補多少貨

✅ 好的寫法：
補貨數量計算：
1. 目標庫存 = 安全庫存量 × 2
2. 需求量 = 目標庫存 - 可用庫存
3. 建議補貨數量 = CEILING(需求量 / 最低訂購量) × 最低訂購量
   （向上取到最低訂購量的倍數）
4. 預估金額 = 建議補貨數量 × 單價
```

### 原則 3：條件格式顏色要定義

```
❌ 不好的寫法：
低庫存標紅色

✅ 好的寫法：
條件格式設定：
- 庫存狀態「缺貨」→ 淺紅底 #FFC7CE
- 庫存狀態「低於安全庫存」→ 淺黃底 #FFEB9C
- 庫存狀態「正常」→ 淺綠底 #C6EFCE
```

### 原則 4：採購單格式要專業

```
❌ 不好的寫法：
產生採購單

✅ 好的寫法：
採購單格式：
- 第 1 列：標題「採購單」（16pt 粗體）
- 第 2 列：供應商名稱
- 第 3 列：聯絡人、電話
- 第 4 列：日期
- 第 5 列起：商品明細（商品編號、商品名稱、數量、單價、金額）
- 最後一列：總計金額
```

## 📝 完整 Prompt 範本

### Prompt 1：資料清理

```
【角色】你是資深倉管系統分析師，熟悉庫存管理與 Python 程式設計

【情境】
- 輸入檔案：raw/inventory_detail.xlsx（庫存明細）
- 輸出目錄：output/

【任務】資料清理

1. 庫存數量修正
   - 轉為數值型態
   - 負數修正為 0

2. 已預留數量修正
   - 若已預留數量 > 庫存數量，修正為庫存數量

3. 商品編號標準化
   - 全部轉大寫
   - 移除空格

4. 日期格式標準化
   - 統一為 YYYY/MM/DD

【輸出】
- cleaned_inventory.xlsx：清理後庫存
- cleaning_log.xlsx：清理日誌

【驗證】顯示清理修正的總筆數
```

### Prompt 2：補貨建議

```
【角色】你是資深倉管系統分析師

【情境】
- 庫存資料：output/cleaned_inventory.xlsx
- 商品主檔：raw/product_master.xlsx
- 輸出：output/reorder_suggestions.xlsx

【任務】產生補貨建議

1. 彙總各商品的總庫存（依商品編號 GROUP BY，SUM 庫存數量）

2. 計算可用庫存 = 總庫存 - 總預留

3. 與商品主檔 LEFT JOIN（取得安全庫存量、供應商）

4. 庫存水位判斷：
   - 可用庫存 ≤ 0 → 「缺貨」
   - 可用庫存 < 安全庫存量 → 「低於安全庫存」
   - 可用庫存 < 安全庫存量 × 1.5 → 「庫存偏低」
   - 其他 → 「正常」

5. 計算建議補貨數量（僅對「缺貨」和「低於安全庫存」）：
   - 目標 = 安全庫存量 × 2
   - 需求 = 目標 - 可用庫存
   - 建議補貨數量 = CEILING(需求 / 最低訂購量) × 最低訂購量

6. 預估金額 = 建議補貨數量 × 單價

【輸出欄位】
商品編號、商品名稱、類別、供應商名稱、可用庫存、安全庫存量、庫存狀態、建議補貨數量、預估金額

【格式】
- 標題列：深藍底、白字
- 缺貨 → 淺紅底
- 低於安全庫存 → 淺黃底
- 金額欄：千分位格式

【驗證】顯示缺貨商品數、低於安全庫存商品數
```

## 💡 常見錯誤與修正

### 錯誤 1：負數庫存沒處理

**問題**：補貨計算結果為負數

**原因**：原始資料有負數庫存

**修正**：先將負數庫存修正為 0

### 錯誤 2：最低訂購量沒考慮

**問題**：建議補貨數量是 7，但最低訂購量是 10

**原因**：直接用需求量作為建議量

**修正**：使用 `CEILING(需求 / 最低訂購量) × 最低訂購量`

### 錯誤 3：已預留數量邏輯錯誤

**問題**：可用庫存變成負數

**原因**：已預留數量大於庫存數量

**修正**：清理階段就修正此問題
