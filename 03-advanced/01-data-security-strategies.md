# 教學一：使用 AI 處理 Excel 時的資料安全策略

> 適用對象：所有使用 AI 工具處理含敏感資料 Excel 的工作者
> 教學目標：學會在使用 AI 自動化 Excel 前，先做好資料脫敏與保護，避免洩漏個資或機密

---

## 一、為什麼這件事很重要？

### 現實風險

當你把 Excel 檔案交給 AI 工具處理時，你可能不經意地把以下資訊暴露出去：

- **個人資料**：身分證字號、手機號碼、地址、銀行帳號
- **商業機密**：薪資明細、客戶名單、合約金額、成本結構
- **醫療資訊**：病歷、健檢報告、保險理賠紀錄

即使你使用的是本機執行的 AI 工具（如 Claude Code CLI），養成「先脫敏再處理」的習慣仍然很重要——因為：

1. 程式碼可能被版本控制系統（Git）記錄，不小心推送到公開儲存庫
2. 開發過程中的螢幕截圖、Log 檔可能包含真實資料
3. 未來可能需要把同一份 Prompt 分享給同事使用

### 核心原則

> **能用假資料開發的，就不要用真資料。**
> **必須用真資料的，先脫敏再處理。**

## 二、資料脫敏的三種層級

### 層級一：完全替換（推薦用於開發測試）

用程式產生結構相同但內容完全虛構的假資料。這是最安全的做法。

| 真實欄位 | 假資料替換方式 |
|----------|--------------|
| 身分證字號 | 隨機產生合法格式（1英文+9數字） |
| 姓名 | 從常見姓氏+名字組合隨機產生 |
| 手機號碼 | `09XX-XXX-XXX` 隨機填入 |
| 銀行帳號 | 隨機 10-14 碼數字 |
| 地址 | 從縣市+路名+門牌號隨機組合 |
| Email | `user{流水號}@example.com` |

### 層級二：部分遮蔽（適用於需要保留部分資訊的場景）

保留欄位結構和部分特徵，但遮蔽關鍵識別資訊。

| 欄位 | 原始值 | 遮蔽後 |
|------|--------|--------|
| 身分證字號 | A123456789 | A12345\*\*\*\* |
| 手機 | 0912-345-678 | 0912-\*\*\*-\*\*\* |
| 姓名 | 王小明 | 王○○ |
| Email | john@company.com | j\*\*\*@company.com |

### 層級三：欄位移除（適用於該欄位完全不需要的場景）

直接把敏感欄位從資料中刪除，只保留分析需要的欄位。

## 三、實作示範：假資料生成腳本

### 3.1 AI Prompt（可直接複製使用）

以下 Prompt 讓 AI 幫你產生一份結構與真實資料相同、但內容完全虛構的假資料：

```
請建立 generate_fake_data.py，產生一份模擬的員工薪資明細 Excel 檔案。

需求：
1. 產生 50 筆員工資料
2. 欄位：
   - 員工編號（EMP+4碼流水號）
   - 姓名（從常見中文姓氏+名字隨機組合）
   - 身分證字號（合法格式：1碼大寫英文+9碼數字，隨機產生）
   - 部門（從「人資部、財務部、研發部、業務部、行銷部」隨機選取）
   - 職等（從「專員、副理、經理、協理、副總」隨機選取）
   - 月薪（30000-150000 之間的隨機整數，千位數取整）
   - 手機（09 開頭的 10 碼數字）
   - Email（姓名拼音@example.com 格式）
   - 到職日（2015-2024 年間的隨機日期）
   - 銀行帳號（隨機 12 碼數字）

3. 同時產生一份「遮蔽版」：
   - 身分證字號只保留前 4 碼，後面用 * 替代
   - 手機只保留前 4 碼
   - 銀行帳號只保留後 4 碼
   - Email 只保留 @ 後面的域名

4. 輸出兩份檔案到 demo/ 目錄：
   - fake_salary_full.xlsx（完整版，僅供本機測試）
   - fake_salary_masked.xlsx（遮蔽版，可安全分享）

5. 在終端機顯示遮蔽版的前 5 筆資料作為預覽
```

### 3.2 腳本解說

上面這個 Prompt 的設計重點：

1. **明確指定每個欄位的產生規則**：不是說「產生假資料」，而是逐欄說明格式
2. **同時產出兩個版本**：開發用完整版 + 可分享的遮蔽版
3. **要求預覽**：立即驗證產出是否正確

實作程式碼請參見 `demo/generate_fake_data.py`。

## 四、安全檢查清單

在把任何 Excel 交給 AI 處理之前，請逐一確認：

### 提交前檢查

- [ ] 檔案中是否包含真實的身分證字號？→ 替換為假資料或遮蔽
- [ ] 檔案中是否包含真實的姓名？→ 如非必要，替換為代號
- [ ] 檔案中是否包含銀行帳號或信用卡號？→ 遮蔽或移除
- [ ] 檔案中是否包含手機號碼或地址？→ 遮蔽或移除
- [ ] 檔案中是否包含商業機密（合約金額、成本結構）？→ 評估是否需要脫敏
- [ ] 你的 Prompt 中是否引用了真實資料的內容？→ 改用假資料範例

### 開發完成後檢查

- [ ] Git 記錄中是否有真實資料的檔案？→ 用 `.gitignore` 排除
- [ ] Log 檔或輸出中是否印出了敏感欄位？→ 修改程式只印出非敏感欄位
- [ ] 程式碼中是否硬編碼了真實資料？→ 改用環境變數或設定檔

### .gitignore 範例

```
# 排除所有可能包含真實資料的檔案
*.xlsx
*.csv
!**/raw/*.xlsx
!**/output/*.xlsx

# 排除環境變數
.env
*.env

# 排除 Log
*.log
```

## 五、進階技巧：自動化脫敏流程

### 5.1 Prompt：批次脫敏腳本

```
請寫一個 Python 腳本 batch_mask.py，功能是：

讀取指定的 Excel 檔案，根據設定檔自動遮蔽敏感欄位。

設定格式（寫在腳本開頭的字典）：
MASK_RULES = {
    "身分證字號": {"keep_first": 4, "mask_char": "*"},
    "手機": {"keep_first": 4, "mask_char": "*"},
    "銀行帳號": {"keep_last": 4, "mask_char": "*"},
    "Email": {"mask_type": "domain_only"},  # 只保留 @domain
    "姓名": {"mask_type": "surname_only"},  # 只保留姓氏，名字替換為○
}

使用方式：
python batch_mask.py input.xlsx output_masked.xlsx

如果欄位名稱在 MASK_RULES 中有對應規則，就套用遮蔽；沒有的欄位則保持原樣。
完成後顯示被遮蔽的欄位數量。
```

### 5.2 最佳實踐流程

```
                    ┌─────────────────┐
                    │  取得真實資料     │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  執行脫敏腳本     │
                    │  batch_mask.py   │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
    ┌─────────▼──────┐ ┌────▼────┐ ┌──────▼─────────┐
    │ 完整版（本機）    │ │ 遮蔽版   │ │ 假資料版        │
    │ 不進 Git        │ │ 可分享   │ │ 進 Git 當範例   │
    └────────────────┘ └─────────┘ └────────────────┘
```

## 六、常見問題

### Q: 用 Claude Code 的本機模式是否安全？

Claude Code CLI 在本機執行 Python 腳本時，資料不會上傳到雲端。但你的 Prompt 會送到 API。
因此：**Prompt 中不要包含真實資料**。把真實資料放在本機檔案中，Prompt 中只寫「讀取 xxx.xlsx」。

### Q: 如果 AI 工具需要看到資料才能寫程式，怎麼辦？

先用假資料讓 AI 寫完程式並測試通過，確認邏輯正確後，再把輸入換成真實資料在本機執行。

### Q: 怎麼確保 AI 產生的假資料夠「真實」？

在 Prompt 中指定：
1. 資料分布（例如薪資 30000-150000）
2. 格式規範（例如身分證 1 英文+9 數字）
3. 業務規則（例如到職日不能早於出生日）
4. 類別比例（例如部門大致均分）

## 七、本章重點摘要

1. **先脫敏再處理** — 養成習慣，不要等出事才後悔
2. **三種脫敏層級** — 完全替換 > 部分遮蔽 > 欄位移除，依場景選擇
3. **用假資料開發** — Prompt 搭配 `generate_fake_data.py` 產生安全的測試資料
4. **自動化脫敏** — 用腳本批次處理，不要手動遮蔽（會漏）
5. **Git 安全** — `.gitignore` 排除真實資料，程式碼不硬編碼敏感資訊
