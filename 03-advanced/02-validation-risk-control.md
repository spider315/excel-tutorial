# 教學二：AI 產出的 Excel 驗證與風險控制

> 適用對象：使用 AI 自動化 Excel 報表的業務人員、主管
> 教學目標：學會驗證 AI 產出的數據正確性，建立系統化的品質檢查機制

---

## 一、為什麼需要驗證 AI 的產出？

### AI 不是萬能的

AI 產生的程式碼可能有以下問題：

| 問題類型 | 範例 | 後果 |
|----------|------|------|
| **計算錯誤** | 用錯彙總方式（AVG 當 SUM） | 報表數字偏差 |
| **合併錯誤** | LEFT JOIN 寫成 INNER JOIN | 少了一些紀錄 |
| **遺漏資料** | 沒處理 NaN / 空值 | 統計數字不準 |
| **邏輯反轉** | 升序寫成降序 | 排名顛倒 |
| **欄位錯配** | 把「成本」當「售價」 | 利潤算錯 |
| **型態錯誤** | 文字型數字未轉換 | 加總變成字串拼接 |

### 核心觀念

> **AI 產出的第一版，永遠要驗證。**
> **不是因為 AI 不可信，而是因為你的 Prompt 可能有歧義。**

驗證不是在質疑 AI，而是在確認「你的需求是否被正確理解和執行」。

## 二、四層驗證框架

### 第一層：結構驗證（檔案層級）

確認產出的檔案結構是否正確。

```
✅ 檢查項目：
- 檔案數量是否正確？（預期 8 個檔案，實際幾個？）
- 每個 Excel 的工作表數量和名稱是否正確？
- 每個工作表的欄位名稱是否正確？
- 每個工作表的資料筆數是否合理？
```

**AI Prompt（驗證用）：**
```
請讀取 output/ 目錄下的所有 Excel 檔案，列出：
1. 每個檔案的工作表名稱
2. 每個工作表的欄位名稱
3. 每個工作表的資料筆數
4. 與預期的對照表比較，標記不一致之處
```

### 第二層：數值驗證（抽樣檢查）

從關鍵數字進行抽樣驗證。

```
✅ 檢查項目：
- 銷售總額 = 各月加總 = 各部門加總？（交叉驗證）
- 達成率 = 實際 / 目標？（用 Excel 手算一筆對照）
- 排名順序是否正確？（第 1 名的銷售額確實最高？）
- 百分比加總是否為 100%？
- 毛利 = 銷售額 - 成本？（抽一筆驗算）
```

**AI Prompt（驗證用）：**
```
請對 sales_analysis_report.xlsx 執行以下驗證：

1. 交叉驗證：月度趨勢的銷售總額加總 是否等於 部門樞紐的年度合計加總？
2. 排名驗證：業務員排名 Sheet 的第 1 名銷售總額 是否確實是最高的？
3. 佔比驗證：產品類別佔比加總是否約為 100%？
4. 抽樣驗證：隨機抽取 3 位業務員，從原始銷售明細重新計算其銷售總額，
             與報表中的數字比對是否一致

顯示所有驗證結果，不一致的地方標記 ✘。
```

### 第三層：邏輯驗證（業務規則）

確認產出符合業務邏輯。

```
✅ 檢查項目：
- 達成率是否有超過 500% 的異常值？（可能是資料錯配）
- 毛利率是否在合理範圍（10%-70%）？（超出可能是成本欄位錯）
- 有沒有業務員出現在銷售報表但不在預算表中？
- 月成長率是否有超過 ±50% 的極端值？（需要人工確認）
```

### 第四層：格式驗證（視覺檢查）

打開 Excel 檔案，用肉眼確認格式。

```
✅ 檢查項目：
- 標題列的底色、字體是否正確？
- 條件格式是否有作用？（綠/黃/紅底色是否正確顯示？）
- 圖表的標題、軸標籤是否正確？
- 欄寬是否適當？（有沒有被截斷的文字？）
- 數字是否有千分位逗號？
- 日期格式是否統一？
```

## 三、實作示範：自動化驗證腳本

### 3.1 AI Prompt（可直接複製使用）

```
請建立 formula_validator.py，
讀取 03-advanced/output/ 目錄下的報表檔案，執行以下自動化驗證：

驗證 1 — 結構檢查：
列出所有 xlsx 檔案的工作表名稱與欄位數、資料筆數

驗證 2 — 銷售交叉驗證：
讀取 sales_analysis_report.xlsx：
- 月度銷售趨勢 Sheet 的「銷售總額」欄位加總 = total_A
- 部門銷售樞紐 Sheet 的「年度合計」欄位加總 = total_B
- 比較 total_A 和 total_B 是否一致（允許 ±1 誤差）

驗證 3 — KPI 公式檢查：
讀取 kpi_dashboard.xlsx 的「個人KPI達成率」Sheet：
- 抽取每一筆：驗證「達成率%」是否等於「實際銷售額 / 年度目標金額 × 100」
- 驗證「達成狀態」是否與達成率的分級規則一致

驗證 4 — 利潤公式檢查：
讀取 product_profit_report.xlsx：
- 抽取每一筆：驗證「毛利率%」是否等於「估計毛利 / 銷售總額 × 100」

驗證 5 — 完整性檢查：
- 清理日誌筆數是否 > 0
- 資料品質問題筆數是否 > 0
- 所有預期的輸出檔案是否都存在

每項驗證顯示 ✔（通過）或 ✘（失敗）及詳細原因。
最後顯示整體通過率。
```

### 3.2 腳本解說

這個驗證腳本的設計重點：

1. **四個層面的自動化檢查**：結構、數值、邏輯、完整性
2. **允許微小誤差**：浮點數計算可能有精度差異，設定 ±1 的容忍範圍
3. **逐筆公式驗算**：不是只看總數，而是每一筆都重新算一次
4. **明確的通過/失敗標記**：用 ✔ / ✘ 一目了然

實作程式碼請參見 `demo/formula_validator.py`。

## 四、驗證 Prompt 的撰寫技巧

### 技巧一：指定驗算公式

```
❌ 確認數字是否正確

✅ 驗證：達成率% 是否等於 (實際銷售額 / 年度目標金額) × 100
   允許 ±0.1 的浮點誤差
```

### 技巧二：要求交叉驗證

```
❌ 檢查銷售總額

✅ 從三個不同角度驗證銷售總額是否一致：
   角度 A：月度趨勢表各月加總
   角度 B：部門樞紐表各部門加總
   角度 C：業務員排名表所有人加總
   三個數字應該相等
```

### 技巧三：指定容忍範圍

```
❌ 數字要一模一樣

✅ 數值比較允許 ±1 的誤差（浮點數精度）
   百分比比較允許 ±0.1% 的誤差
   筆數比較必須完全一致（整數，不容許誤差）
```

### 技巧四：要求顯示失敗細節

```
❌ 告訴我有沒有通過

✅ 每項驗證顯示：
   - ✔ 通過：簡短確認訊息
   - ✘ 失敗：預期值、實際值、差異量
   最後統計：通過 X 項 / 總計 Y 項
```

## 五、風險分級與處理策略

### 風險矩陣

| 風險等級 | 情境 | 處理方式 |
|----------|------|----------|
| **低** | 內部參考用的趨勢圖 | 目視確認圖表趨勢方向正確即可 |
| **中** | 部門月會報告 | 抽樣驗證 3-5 筆關鍵數字 |
| **高** | 提交給高階主管的 KPI 報表 | 完整自動化驗證 + 人工複核 |
| **極高** | 對外公開或法規要求的財務報表 | 雙人審核 + 源頭資料比對 |

### 處理流程

```
AI 產出報表
     │
     ▼
┌──────────────────┐
│ 1. 執行驗證腳本    │ ← formula_validator.py
└────────┬─────────┘
         │
    全部通過？
    ┌──┴──┐
   是      否
    │      │
    ▼      ▼
 人工抽查  檢視失敗項目
    │      │
    ▼      ▼
  交付    修正 Prompt
         重新產出
           │
           └──→ 回到步驟 1
```

## 六、常見的 AI 產出問題與排查

### 問題一：加總數字對不上

**可能原因：**
- 有些列被過濾掉了（NaN、負數）
- JOIN 方式不對（INNER vs LEFT）
- 重複資料未去除

**排查 Prompt：**
```
請比較 sales_analysis_report.xlsx 的月度總額加總
與 cleaned_monthly_sales.xlsx 的銷售金額加總，
如果不一致，請列出被排除的資料（金額 ≤ 0 或數量 ≤ 0 的筆數和總額）。
```

### 問題二：排名順序錯誤

**可能原因：**
- ascending 參數方向反了
- 排序欄位選錯

**排查 Prompt：**
```
讀取業務員排名 Sheet，
顯示排名第 1、2、3 名的業務員和銷售總額，
確認第 1 名的總額 ≥ 第 2 名 ≥ 第 3 名。
```

### 問題三：條件格式沒有作用

**可能原因：**
- 欄位是文字型，不是數值型
- 條件格式的範圍設錯
- 色碼寫錯

**排查 Prompt：**
```
讀取 kpi_dashboard.xlsx，
檢查「達成率%」欄位的資料型態（應為數值），
列出條件格式規則的範圍和條件。
```

### 問題四：毛利率出現負數或超過 100%

**可能原因：**
- 成本欄位和售價欄位搞混
- 銷售金額包含折扣，低於成本

**排查 Prompt：**
```
讀取 product_profit_report.xlsx，
列出毛利率 < 0 或 > 100 的產品，
顯示其銷售總額、成本、毛利的計算過程。
```

## 七、建立團隊驗證規範

### 建議的團隊流程

1. **Prompt 審核制度**：重要報表的 Prompt 由兩人審核
2. **驗證腳本標準化**：每種報表都有對應的驗證腳本
3. **產出版本控制**：每次產出保留日期標記（如 `report_20240315.xlsx`）
4. **錯誤紀錄**：記錄每次驗證發現的問題，作為改進 Prompt 的依據

### 驗證腳本的命名規範

```
validate_{報表名稱}.py

例如：
validate_sales_report.py
validate_kpi_dashboard.py
validate_monthly_summary.py
```

## 八、本章重點摘要

1. **四層驗證**：結構 → 數值 → 邏輯 → 格式，由粗到細
2. **自動化驗證**：用 `formula_validator.py` 自動檢查公式和交叉驗證
3. **指定公式**：驗證 Prompt 中寫明預期的計算方式
4. **容忍誤差**：浮點數允許 ±1，百分比允許 ±0.1%
5. **風險分級**：越重要的報表，驗證越嚴格
6. **排查套路**：數字對不上 → 檢查過濾條件和 JOIN 方式
