# 進階 AI Prompt 教學：多維度銷售分析與精美 Excel 報表

> 適用對象：已完成基礎教學的業務分析人員、主管
> 前置條件：建議先完成 `hr_demo/TUTORIAL.md` 的基礎教學
> 教學目標：學會用 AI 完成「多檔案交叉分析 → 樞紐報表 → 圖表視覺化 → 條件格式」的完整流程

---

## 一、前言：從基礎清理到進階分析

### 基礎 vs 進階的差異

在基礎教學中，我們學會了：
- 單一檔案的格式清理
- 兩份檔案之間的比對
- 簡單的摘要統計

但在真實工作中，你經常面對更複雜的需求：

| 基礎場景 | 進階場景 |
|----------|----------|
| 清理單一欄位的格式 | 跨檔案的名稱比對與標準化 |
| 簡單的加總統計 | 多維度樞紐分析（部門 × 月份 × 產品） |
| 純文字報表 | 含圖表、條件格式的精美 Excel 報表 |
| 找出異常值 | 趨勢分析、成長率計算、KPI 達成率 |
| 靜態資料比對 | 預算 vs 實際的差異分析 |

### 本教學涵蓋的進階技能

1. **多檔案交叉分析** — 四份檔案的關聯與合併
2. **樞紐分析表** — 用 Prompt 指定行列維度與彙總方式
3. **趨勢與成長率計算** — 月度 MoM 成長率、排名
4. **KPI 達成率儀表板** — 預算 vs 實際比較
5. **產品利潤與滿意度交叉分析** — 合併不同來源的指標
6. **精美報表格式** — 條件格式、資料條、圖表（折線圖、長條圖、圓餅圖）

## 二、完整案例說明

### 情境描述

你是一家消費電子公司的業務分析師。年底到了，老闆要你整理全年度的銷售分析報告，包含業績排名、預算達成率、產品利潤分析等。

你手上有四份 Excel 檔案：

| 檔案 | 說明 | 筆數 |
|------|------|------|
| `monthly_sales.xlsx` | 全年度銷售明細 | ~600 筆 |
| `budget_targets.xlsx` | 每位業務的年度預算目標 | 30 筆 |
| `product_catalog.xlsx` | 產品目錄（含成本） | 15 筆 |
| `customer_feedback.xlsx` | 客戶滿意度回饋 | 200 筆 |

### 資料中的品質問題

跟真實世界一樣，這些資料中有各種品質問題：

- 銷售金額出現負數（退貨未標記）
- 日期格式不統一（`-` vs `/`）
- 產品名稱錯字（「智彗」→「智慧」）、多餘空白
- 業務員姓名不一致（「張冠宇」vs「張冠宇（代）」）
- 回饋評分超出 1-5 範圍（出現 0、6、-1）
- 重複訂單編號
- 銷售數量為 0

### 目標產出

```
四份原始 Excel
    ↓ 任務一：資料清理 + 清理日誌
清理後檔案（3 份）+ 清理日誌（1 份）
    ↓ 任務二：多維度銷售分析
銷售分析報表（含 5 個工作表 + 圖表）
    ↓ 任務三：KPI 達成率分析
KPI 儀表板（含條件格式 + 圖表）
    ↓ 任務四：產品利潤交叉分析
產品利潤報表（含資料條）
    ↓ 任務五：資料品質報告
品質問題清單 + 統計 + 修正日誌
```

## 三、Prompt 撰寫教學（進階技巧）

### 3.1 進階 Prompt 的六大設計原則

#### 原則一：分層描述——先概覽、再細節

進階任務通常涉及多個分析維度。先告訴 AI「整體目標」，再逐層展開細節。

```
❌ 不好的寫法：
按部門統計銷售額，然後按月份統計，然後按產品統計，然後做排名，然後算成長率...

✅ 好的寫法：
任務二：多維度銷售分析

請從以下五個維度分析銷售數據：
2-1. 月度銷售趨勢（含 MoM 成長率）
2-2. 部門銷售樞紐分析（部門 × 月份矩陣）
2-3. 業務員排名（Top 10，含訂單數與銷售總額）
2-4. 產品類別銷售分布（含佔比）
2-5. 客戶區域銷售分布
```

**為什麼？** 先列出五個維度的「大標題」，AI 就能掌握全貌。每個維度再各自展開細節，不會遺漏。

---

#### 原則二：明確指定彙總方式——SUM、AVG、COUNT 不要混淆

```
❌ 不好的寫法：
統計各部門的銷售情況

✅ 好的寫法：
按部門彙總：
- 銷售筆數（COUNT）
- 銷售總額（SUM of 銷售金額）
- 平均每單金額（AVG of 銷售金額）
- 按月份展開成樞紐表（行=部門，列=月份，值=SUM 銷售金額）
```

**為什麼？** 「銷售情況」太模糊——是要總額、筆數、平均值、還是全部都要？明確指定彙總函數，結果才會精準。

---

#### 原則三：指定計算公式——避免 AI 自己推導

```
❌ 不好的寫法：
計算成長率

✅ 好的寫法：
月成長率（MoM%）= (本月銷售總額 - 上月銷售總額) / 上月銷售總額 × 100
達成率（%）= 實際銷售額 / 年度目標金額 × 100
毛利率（%）= (銷售總額 - 估計成本總額) / 銷售總額 × 100
```

**為什麼？** 成長率有很多種算法（MoM、YoY、CAGR……）。不指定公式，AI 可能用了你不想要的算法。

---

#### 原則四：用條件分級——讓 AI 自動標記狀態

```
❌ 不好的寫法：
標記哪些業務表現好

✅ 好的寫法：
根據達成率標記狀態：
- 達成率 ≥ 120%：★ 超標達成（綠底）
- 100% ≤ 達成率 < 120%：✔ 達成（綠底）
- 80% ≤ 達成率 < 100%：△ 接近達成（黃底）
- 達成率 < 80%：✘ 未達成（紅底）
```

**為什麼？** 「表現好」的標準每個人不同。寫出明確的數值區間與對應標籤，AI 才能準確分類。

---

#### 原則五：指定視覺化類型——不讓 AI 自選圖表

```
❌ 不好的寫法：
加幾個圖表

✅ 好的寫法：
在「月度銷售趨勢」工作表中加入：
1. 折線圖：X 軸=月份，Y 軸=銷售總額，標題「2024 年月度銷售趨勢」
2. 長條圖：X 軸=月份，Y 軸=訂單數，標題「月度訂單數量」
圖表寬度 25，高度 14，放置於 A15 和 A32 位置

在「產品類別分布」工作表中加入：
3. 圓餅圖：顯示各類別銷售佔比，標題「產品類別銷售佔比」
```

**為什麼？** 折線圖適合看趨勢、長條圖適合比較大小、圓餅圖適合看佔比。告訴 AI 你要哪種圖表，才不會得到不適合的視覺化方式。

---

#### 原則六：指定 Excel 格式化細節——標題色、條件格式、欄寬

```
❌ 不好的寫法：
把報表弄漂亮一點

✅ 好的寫法：
Excel 格式要求：
- 標題列：粗體白字、深藍底色 (#2F5496)、置中對齊
- 所有儲存格：細邊框、垂直置中
- 欄寬自動調整（中文字算 2 個字元寬）
- 達成率欄位加條件格式：≥100 綠底、80-99 黃底、<80 紅底
- 毛利率欄位加資料條（藍色漸層）
- 業務排名 Top 3 加淺綠底色
```

**為什麼？** 「漂亮」是主觀的。你心中的「專業報表」跟 AI 預設的可能差很遠。把色碼、格式都寫清楚，產出才是你要的樣子。

### 3.2 逐段拆解完整 Prompt

以下是本教學實際使用的 Prompt，逐段說明設計思路。

---

**【開頭：指定架構與路徑】**

```
請在 03-advanced/ 目錄下建立 advanced_analysis.py，
讀取 03-advanced/raw/ 的四份原始 Excel：
- monthly_sales.xlsx（月度銷售明細，約 600 筆）
- budget_targets.xlsx（年度預算目標，30 筆）
- product_catalog.xlsx（產品目錄，15 筆）
- customer_feedback.xlsx（客戶回饋，200 筆）

處理結果輸出到 03-advanced/output/ 目錄。
```

> **設計重點：** 標注每份檔案的大致筆數，幫助 AI 預估資料規模和處理策略。

---

**【任務一：資料清理 — 含日誌追蹤】**

```
任務一：資料清理與標準化

清理規則：
1. 日期格式：統一為 YYYY-MM-DD（/ → -）
2. 業務員姓名：去除空白、移除括號標注（如「張冠宇（代）」→「張冠宇」）
3. 產品名稱：去前後空白、修正錯字（「智彗」→「智慧」、「藍芽」→「藍牙」）
4. 數值欄位：轉為數值型態（數量、金額、評分等）

重要：每次修正都要記錄到清理日誌（cleaning_log.xlsx），
欄位：來源檔案、列號、欄位、原始值、修正值、問題描述

輸出：
- cleaned_monthly_sales.xlsx
- cleaned_budget_targets.xlsx
- cleaned_customer_feedback.xlsx
- cleaning_log.xlsx
```

> **進階亮點：** 與基礎教學不同，這裡要求產出「清理日誌」——這是企業級資料處理的標準做法，讓每一次修改都有紀錄可追溯。

---

**【任務二：多維度銷售分析】**

```
任務二：多維度銷售分析 → sales_analysis_report.xlsx

前置處理：排除銷售金額 ≤ 0 或數量 ≤ 0 的異常紀錄

Sheet 1「月度銷售趨勢」
欄位：月份, 訂單數, 銷售總額, 平均客單價, 月成長率%
公式：月成長率% = (本月銷售總額 - 上月銷售總額) / 上月銷售總額 × 100
圖表：
- 折線圖（銷售總額 vs 月份），放 A15
- 長條圖（訂單數 vs 月份），放 A32

Sheet 2「部門銷售樞紐」
行 = 部門，列 = 月份（YYYY-MM），值 = SUM(銷售金額)
最後一欄加「年度合計」

Sheet 3「業務員排名」
欄位：排名, 業務員, 部門, 訂單數, 銷售總額, 平均每單金額
按銷售總額降序排列
Top 3 加淺綠底色

Sheet 4「產品類別分布」
欄位：產品類別, 銷售筆數, 銷售總額, 銷售佔比
圖表：圓餅圖（銷售佔比），放 A10

Sheet 5「客戶區域分布」
欄位：客戶區域, 銷售筆數, 銷售總額
按銷售總額降序
```

> **設計重點：**
> 1. 先指定「前置處理」（過濾無效資料），再開始分析
> 2. 每個 Sheet 都明確指定欄位、排序方式、圖表類型與位置
> 3. 樞紐表的行列維度用「行 = ○○，列 = ○○」的語法，AI 一看就懂

---

**【任務三：KPI 達成率分析】**

```
任務三：預算達成率分析 → kpi_dashboard.xlsx

Sheet 1「個人 KPI 達成率」
步驟：
1. 從清理後的銷售明細，按業務員計算實際銷售總額
2. 與 budget_targets.xlsx 的年度目標金額做合併（以業務員姓名為 key）
3. 計算：達成率% = 實際銷售額 / 年度目標金額 × 100
4. 計算：差異 = 實際銷售額 - 年度目標金額
5. 依據達成率標記狀態：
   - ≥ 120%：★ 超標達成
   - ≥ 100%：✔ 達成
   - ≥ 80%：△ 接近達成
   - < 80%：✘ 未達成

欄位：業務員, 部門, 年度目標金額, 實際銷售額, 達成率%, 差異, 達成狀態
按達成率降序排列

條件格式：
- 達成率 ≥ 100：綠底 (#C6EFCE)
- 80 ≤ 達成率 < 100：黃底 (#FFEB9C)
- 達成率 < 80：紅底 (#FFC7CE)

Sheet 2「部門 KPI 彙總」
欄位：部門, 業務人數, 目標合計, 實際合計, 部門達成率%
圖表：長條圖（部門達成率），放 A10
```

> **設計重點：**
> 1. 用「步驟 1、2、3…」描述計算流程，AI 不會搞混順序
> 2. 條件格式直接給色碼（`#C6EFCE`），不會出現「AI 覺得的綠色」跟你想的不一樣

---

**【任務四：產品利潤交叉分析】**

```
任務四：產品利潤與客戶滿意度交叉分析 → product_profit_report.xlsx

Sheet 1「產品利潤與滿意度」
步驟：
1. 從銷售明細按產品編號彙總：銷售數量(SUM)、銷售總額(SUM)
2. 合併產品目錄取得：產品名稱、類別、建議售價、成本
3. 計算：估計成本總額 = 銷售數量 × 成本
4. 計算：估計毛利 = 銷售總額 - 估計成本總額
5. 計算：毛利率% = 估計毛利 / 銷售總額 × 100
6. 合併客戶回饋（排除評分不在 1-5 範圍的）：按產品編號計算平均滿意度、回饋筆數

欄位順序：產品編號, 產品名稱, 產品類別, 銷售數量, 銷售總額,
         建議售價, 成本, 估計成本總額, 估計毛利, 毛利率%,
         平均滿意度, 回饋筆數

按估計毛利降序排列
毛利率欄位加資料條（藍色漸層 #5B9BD5）
```

> **設計重點：** 這是最典型的「交叉分析」——把銷售、成本、客戶回饋三個來源的指標合到同一張表。步驟化描述讓 AI 知道先算什麼、再合併什麼。

---

**【任務五：資料品質報告】**

```
任務五：資料品質報告 → data_quality_report.xlsx

Sheet 1「資料品質問題清單」
掃描原始資料，找出以下問題：

- 銷售數量為 0
- 訂單編號重複
- 客戶回饋評分超出 1-5 範圍
- 預算 Q1-Q4 合計超過年度目標的 115%
欄位：來源檔案, 列號, 問題類型, 欄位, 原始值, 問題描述

Sheet 2「問題統計」
按「問題類型 × 來源檔案」彙總筆數

Sheet 3「清理修正日誌」
直接使用任務一產出的 cleaning_log 資料
```

### 3.3 完整的 Prompt（可直接複製使用）

#### Prompt 1：資料生成（產生練習用模擬資料）

```
請在 03-advanced/ 目錄下建立 generate_data.py，產生四份模擬的銷售分析 Excel 檔案，
存放到 03-advanced/raw/ 目錄。

檔案一：monthly_sales.xlsx（月度銷售明細）
- 30 名業務員 × 12 個月（2024年），每人每月 1-3 筆訂單
- 欄位：訂單編號(ORD-00001格式)、訂單日期、業務員、部門、
        產品編號、產品名稱、數量、單價、銷售金額、客戶區域
- 5 個部門：北區業務部、中區業務部、南區業務部、海外業務部、電商部
- 15 項產品（穿戴裝置、音訊設備、配件、電腦周邊、家電）

檔案二：budget_targets.xlsx（年度預算目標，30 筆）
欄位：業務員編號、業務員、部門、年度目標金額、Q1目標、Q2目標、Q3目標、Q4目標

檔案三：product_catalog.xlsx（產品目錄，15 筆）
欄位：產品編號、產品名稱、產品類別、建議售價、成本、庫存量、上架日期

檔案四：customer_feedback.xlsx（客戶回饋，200 筆）
欄位：回饋編號、日期、產品編號、產品名稱、滿意度評分(1-5)、評語、客戶區域

請在資料中刻意混入以下髒資料：
銷售明細：負數金額(5筆)、日期用/(3筆)、產品名稱錯字(2筆)與多空白(4筆)、
         業務員姓名多空格(3筆)、重複訂單編號(2筆)、數量為0(3筆)
預算目標：業務員姓名有多餘標注(2筆)、Q1-Q4加總超過年度目標120%(1筆)
客戶回饋：評分超出範圍(3筆)、日期用/(2筆)、產品名稱與目錄不一致(2筆)
```

## 四、Prompt 進階技巧——場景範例

### 4.1 如何要求「樞紐分析表」

樞紐分析是 Excel 使用者最常見的需求之一。跟 AI 說明時，用以下格式最清楚：

```
建立樞紐分析表：
- 行（Row）：部門
- 列（Column）：月份
- 值（Value）：SUM(銷售金額)
- 加一欄「年度合計」= 各月加總
- 加一列「全公司合計」= 各部門加總
```

### 4.2 如何要求「條件格式」

條件格式讓報表一目了然。Prompt 要包含三要素：

1. **套用範圍**：哪個欄位的第幾列到第幾列
2. **判斷條件**：大於、等於、介於
3. **格式效果**：底色色碼、字體顏色

```
對「達成率%」欄位（D2:D31）加條件格式：
- 值 ≥ 100 → 底色 #C6EFCE（淺綠）
- 值介於 80~99 → 底色 #FFEB9C（淺黃）
- 值 < 80 → 底色 #FFC7CE（淺紅）
```

### 4.3 如何要求「圖表」

指定圖表時，至少要說明以下資訊：

```
圖表類型：折線圖 / 長條圖 / 圓餅圖 / 堆疊長條圖
X 軸 / 類別：月份
Y 軸 / 數值：銷售金額
標題：「2024 年月度銷售趨勢」
尺寸：寬 25，高 14（openpyxl 單位）
位置：放在工作表的 A15 儲存格
```

### 4.4 如何要求「交叉分析」

交叉分析的核心是「合併（merge）」。一定要說明：

```
合併方式：
1. 先從 monthly_sales 按產品編號 GROUP BY，算 SUM(銷售金額)
2. LEFT JOIN product_catalog（用產品編號），取得成本欄位
3. LEFT JOIN customer_feedback 的平均滿意度（先按產品編號 GROUP BY AVG）
4. 合併後計算毛利率 = (銷售總額 - 成本總額) / 銷售總額 × 100
```

## 五、常見錯誤與改進

### 錯誤一：分析維度不明確

```
❌ 分析銷售數據

✅ 從以下五個維度分析：
   1. 月度趨勢（含 MoM 成長率）
   2. 部門 × 月份樞紐表
   3. 業務員排名 Top 10
   4. 產品類別佔比
   5. 客戶區域分布
```

### 錯誤二：彙總方式不清楚

```
❌ 統計各部門銷售

✅ 按部門計算：
   - COUNT(訂單編號) → 訂單數
   - SUM(銷售金額) → 銷售總額
   - AVG(銷售金額) → 平均客單價
```

### 錯誤三：圖表需求太模糊

```
❌ 加幾個圖表

✅ 折線圖：X=月份, Y=銷售總額, 標題「月度趨勢」, 放 A15, 寬 25 高 14
   圓餅圖：類別=產品類別, 值=銷售佔比, 放 A10
```

### 錯誤四：條件格式沒給色碼

```
❌ 用紅綠燈標記達成率

✅ 達成率條件格式：
   ≥ 100 → #C6EFCE（綠）
   80-99 → #FFEB9C（黃）
   < 80  → #FFC7CE（紅）
```

### 錯誤五：交叉分析沒說合併順序

```
❌ 把銷售和成本和滿意度放在一起

✅ 步驟：
   1. 銷售按產品 GROUP BY SUM
   2. LEFT JOIN 產品目錄取成本
   3. LEFT JOIN 回饋平均分數
   4. 計算毛利率
```

## 七、延伸應用

### 其他業務分析場景

本教學的 Prompt 結構可以套用到：

- **財務月結報表**：多帳戶的收支明細 → 樞紐分析 → 損益表
- **行銷活動分析**：不同渠道的轉換率 → 漏斗圖 → ROI 計算
- **庫存管理**：進出貨明細 → 周轉率 → 安全庫存告警
- **專案進度追蹤**：多專案的里程碑 → 甘特圖式呈現 → 延遲告警

### 如何擴充 Prompt？

你可以在現有 Prompt 基礎上加入：

1. **更多圖表**：堆疊長條圖、散佈圖、組合圖
2. **更多條件格式**：色階（Color Scale）、圖示集（Icon Set）
3. **跨年比較**：同時分析 2023 和 2024 資料，計算 YoY 成長率
4. **自動寄信**：產出報表後自動用 Python 寄出 Email
5. **定時排程**：搭配 crontab 或 Windows 工作排程器，每月自動執行
