# 客戶資料清洗與合併 — AI 指令教學

> **難度**：進階
> **適用對象**：行銷人員、業務人員、CRM 管理員
> **學習目標**：用 AI 合併多來源客戶名單，統一格式並去除重複

---

## 📌 課程情境

**你是行銷部門的數據分析師**，每季要整合來自不同來源的客戶名單：

1. CRM 系統匯出的客戶資料
2. 業務人員維護的 Excel 名單
3. 官網註冊的會員資料

這些名單有各種問題：
- 欄位名稱不同
- 電話格式不一致
- Email 大小寫混亂
- 同一位客戶重複出現

用 AI 可以快速標準化格式、去除重複，產生乾淨的客戶主檔。

## 🎯 Prompt 設計原則

### 原則 1：欄位對應要明確

```
❌ 不好的寫法：
把三份名單合併

✅ 好的寫法：
欄位對應規則：
- CRM：客戶姓名 → 姓名、連絡電話 → 電話、電子郵件 → Email
- 業務Excel：姓名 → 姓名、電話 → 電話、Email → Email
- 網站註冊：full_name → 姓名、phone_number → 電話、email_address → Email
```

### 原則 2：電話清理規則要完整

```
❌ 不好的寫法：
統一電話格式

✅ 好的寫法：
電話標準化為 0912345678 格式：
1. 移除所有非數字字元（-、空格、括號）
2. 國際碼處理：+886912345678 → 0912345678
3. 缺少 0 的補齊：912345678 → 0912345678
4. 中文數字轉換：零→0、一→1、二→2...
5. 最終格式：10 碼數字，以 09 開頭
```

### 原則 3：去重邏輯要明確

```
❌ 不好的寫法：
刪除重複的客戶

✅ 好的寫法：
重複判定規則：
1. 電話相同 → 視為同一客戶
2. Email 相同 → 視為同一客戶
3. 保留策略：保留第一筆（依資料來源優先順序：CRM > 業務Excel > 網站註冊）
4. 重複的記錄另存為「duplicate_records.xlsx」以供人工確認
```

### 原則 4：來源追溯要保留

```
❌ 不好的寫法：
合併成一份名單

✅ 好的寫法：
合併後保留來源資訊：
- 新增欄位「資料來源」，值為 CRM / 業務Excel / 網站註冊
- 保留「原始編號」欄位，對應原始檔案的 ID
```

## 📝 完整 Prompt 範本

### Prompt 1：資料標準化

```
【角色】你是資深數據分析師，熟悉客戶資料管理與 Python 程式設計

【情境】
- 輸入檔案：
  - raw/crm_customers.xlsx（CRM 系統客戶）
  - raw/sales_customers.xlsx（業務 Excel 名單）
  - raw/web_registrations.xlsx（網站註冊）
- 輸出目錄：output/

【任務】標準化三份名單

1. 欄位重新命名：
   - CRM：客戶姓名→姓名、連絡電話→電話、電子郵件→Email、公司名稱→公司
   - 業務Excel：電話→電話、Email→Email、公司→公司
   - 網站註冊：full_name→姓名、phone_number→電話、email_address→Email

2. 電話標準化（目標格式：0912345678）：
   - 移除 -、空格、括號
   - +886 開頭 → 0 開頭
   - 9 開頭（9碼）→ 補 0 變 09 開頭
   - 中文數字 → 阿拉伯數字

3. Email 標準化：
   - 全部轉小寫
   - 驗證格式：必須有 @，@ 後面必須有 .
   - 無效的 Email 清空

4. 姓名標準化：
   - 移除前後空白

5. 新增欄位「資料來源」：CRM / 業務Excel / 網站註冊

【輸出】
- cleaning_log.xlsx：清理日誌（資料來源、欄位、列號、原始值、修正值、原因）

【驗證】顯示各來源的筆數與總清理修正數
```

### Prompt 2：合併與去重

```
【角色】你是資深數據分析師

【情境】
- 輸入：已標準化的三份 DataFrame
- 輸出：output/customer_master.xlsx、output/duplicate_records.xlsx

【任務】合併並去除重複

1. 選取共同欄位：原始編號、姓名、電話、Email、公司、資料來源

2. 垂直合併三份 DataFrame（pd.concat）

3. 重複判定：
   - 電話相同（10碼完整號碼）→ 重複
   - Email 相同 → 重複

4. 標記重複：
   - 新增「重複標記」欄位
   - 值：電話重複 / Email重複 / 電話+Email重複

5. 去重策略：
   - 使用 drop_duplicates，keep="first"
   - 先依電話去重，再依 Email 去重

【輸出】
- customer_master.xlsx：去重後的客戶主檔
- duplicate_records.xlsx：被標記為重複的記錄（供人工確認）

【格式】
- 標題列：深藍底 #2F5496、白字
- 重複記錄列：淺黃底 #FFEB9C

【驗證】顯示合併前總筆數、重複筆數、去重後筆數
```

### Prompt 3：來源統計

```
【角色】你是資深數據分析師

【情境】
- 輸入：output/customer_master.xlsx
- 輸出：output/source_summary.xlsx

【任務】來源統計報表

1. 依「資料來源」分組，計算各來源的客戶數

2. 欄位：資料來源、客戶數

3. 依客戶數降冪排序

【格式】
- 標題列：深藍底、白字
- 數字欄：千分位格式
```

## 💡 常見錯誤與修正

### 錯誤 1：合併後欄位消失

**問題**：某些欄位在合併後變成 NaN

**原因**：三份名單的欄位名稱不一致

**修正**：先統一 rename，再 pd.concat

### 錯誤 2：去重沒有效果

**問題**：明明電話相同，卻沒有被視為重複

**原因**：電話格式不一致（有的有符號，有的沒有）

**修正**：先標準化電話格式，再進行去重

### 錯誤 3：Email 比對失敗

**問題**：同一個 Email 被視為不同

**原因**：大小寫不一致（John@gmail.com vs john@gmail.com）

**修正**：統一轉小寫再比對

## 🚀 進階練習

1. **模糊比對**：姓名相似度超過 80% 且電話後 4 碼相同，視為可能重複

2. **地址標準化**：統一縣市名稱（台北市 vs 臺北市）

3. **資料品質報告**：統計各欄位的空值率、格式錯誤率

4. **增量更新**：與上次合併的主檔比對，只新增新客戶
